理解 __new__ 和 __init__ 在 Python 中的区别：

✅ 一句话总结
方法	作用	比喻
__new__	创建对象实例	像是在“造一辆车”
__init__	初始化实例内容	像是在“给这辆车加油、贴膜”

__new__ 先执行，用来创建对象
__init__ 后执行，用来初始化对象内容

__new__ 是造东西，__init__ 是装修/初始化这个东西。

 四、你写的类默认不写 __new__，是因为…
Python 在大多数情况下，默认使用父类 object.__new__ 来创建实例。只有在以下情况才需要你手动写 __new__：

🔹 需要控制“是否创建”这个对象时，例如：
单例模式（保证只创建一个对象）

继承 str, tuple 这类不可变对象

想拦截或缓存对象创建过程（高级用法）

✅ 五、对比总结表
对比项	__new__	__init__
目的	创建对象（返回实例）	初始化对象（设置属性）
参数第一个是	类（cls）	实例（self）
是否必须返回值	是（必须返回实例）	否（默认返回 None）
什么时候用	想控制对象创建（如单例）	给对象设置属性（常用）
是否总会调用	每次实例化时都调用	每次实例化时都调用
